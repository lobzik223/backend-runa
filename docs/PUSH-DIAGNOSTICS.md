# Диагностика пуш-уведомлений (только сервер)

Приложение на модерации не трогаем. Проверяем только бэкенд и доступ.

## 1. Логирование на бэкенде

После деплоя в коде включено логирование: при успешном сохранении push-токена в логах появится строка:

- `[PUSH] Token registered: userId=... deviceId=... platform=android hasToken=true`
- или `[PUSH] Token updated: ...`

Значит запрос с приложения дошел до сервера и устройство записано в БД.

## 2. Что проверить на сервере

### Запустить просмотр логов бэкенда

```bash
docker logs runa_backend -f
```

Включи в приложении уведомления (Настройки → уведомления вкл.), подожди 5–10 секунд.

- Если в логах появилась строка `[PUSH] Token registered` или `[PUSH] Token updated` — запрос доходит, токен сохраняется. Дальше проверь таблицу:  
  `docker exec runa_postgres psql -U runa -d runa -c 'SELECT id, "userId", platform FROM devices;'`
- Если такой строки нет — запрос до бэкенда не доходит или падает до контроллера.

### Проверить, доходят ли запросы до сервера

Если перед бэкендом стоит nginx / reverse proxy:

```bash
# Пример: доступ к логам nginx (путь может отличаться)
grep -i "push-notifications/token" /var/log/nginx/access.log
# или
docker logs <nginx-container> 2>&1 | grep -i token
```

Увидишь POST на `/api/push-notifications/token` с кодом 200 (успех) или 401 (не авторизован).

### Типичные причины, почему таблица devices пустая

1. **Приложение стучится не на этот сервер**  
   В прод-сборке должен быть твой реальный API (например `https://api.runafinance.online`). Если в приложении зашит другой URL или localhost — запросы пойдут не на этот бэкенд. Менять нужно конфиг сборки приложения и следующую сборку; текущую на модерации не трогаем.

2. **401 Unauthorized**  
   Эндпоинт защищён JWT. Если токен не передаётся или протух — в логах nginx будет 401, в логах бэкенда строки `[PUSH] Token registered/updated` не будет.

3. **Сеть / firewall**  
   Телефон не может достучаться до сервера — тогда запрос вообще не попадёт в логи.

## 3. После появления устройств в БД

Проверка рассылки:

```bash
docker exec runa_backend node dist/scripts/send-push.js --all --title "RUNA Finance" --body "Проверка"
```

Ожидаемый вывод: `Sent to N/M devices (K users)`.
